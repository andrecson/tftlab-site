## Mon Jun 30 2026 - US-024 (Builder: motor de sinergias — funcao pura + testes)
- Implemented the synergy engine as a pure, client-safe module (no imports), plus a full unit-test suite.
- Files changed:
  - `src/lib/synergy.ts` (new): `computeSynergies(units, traits) -> ActiveTrait[]` and the exported helper `countUnitsByTrait(units) -> Map<string,number>`. Types `SynergyUnit { championId, traits: string[] }`, `TraitInfo { key, name, breakpoints: number[] }`, `ActiveTrait { key, name, count, breakpoints, activeBreakpoint, tier, nextBreakpoint, maxed }`.
  - `src/lib/synergy.test.ts` (new): 17 node:test cases.
  - `prd.json` (US-024 passes=true + notes), `progress.txt`.
- Design:
  - PURE + client-safe (lives with tiers.ts/site.ts) so the US-025/026 `"use client"` builder can call it directly. No `db`/Prisma import.
  - KEY-AGNOSTIC: `SynergyUnit.traits` and `TraitInfo.key` share whatever string key the caller uses — catalog by trait NAME, DB by trait id — so the engine is decoupled from the data source (mirrors the badges service's structural-input approach).
  - Board POSITION is deliberately omitted from `SynergyUnit`: TFT trait counts don't depend on where a unit sits, so "positioned units" reduce to `{ championId, traits }` for synergy purposes.
  - UNIQUE COUNTING: dedupe by `championId` (duplicate copy contributes nothing) + `new Set` over each champion's traits (repeated trait counts once). Both satisfy AC "nao duplica a mesma unidade".
  - BREAKPOINTS: normalized (sorted, distinct, positive) defensively; active when count >= first bp; `tier` = highest bp index reached; `nextBreakpoint` null at max. `[]`-breakpoint meta-traits never activate; unknown traits ignored.
  - Output pre-sorted for a synergy panel: tier desc, count desc, name asc.
- Quality: `npm test` -> 24 pass / 0 fail (17 synergy + 7 badges); `npm run typecheck` exit 0; `npm run lint` "No ESLint warnings or errors". Backend/pure-function story — no browser verification in AC.
- **Learnings for future iterations:**
  - See the new Codebase Patterns "Synergy engine (US-024)" line. Most important for US-025/026: import `computeSynergies` from `@/lib/synergy` into the client builder; feed it placed champions mapped to `{ championId, traits }` + the set's trait breakpoints as `TraitInfo[]`.
  - `Trait.breakpoints` is a Prisma `Json` column; when building `TraitInfo[]` from DB rows, cast it (e.g. `bp as number[]`). The catalog's `CatalogTrait.breakpoints` is already `number[]`.
  - Reuse `countUnitsByTrait` for the team unit-count/cost display in US-026 rather than re-deriving unique-unit logic.
